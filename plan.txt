1. プロジェクト概要
既存の求人媒体ページにある情報をベースに、信頼感のあるモダンなコーポレートサイト（1ページ構成のLP形式）としてリニューアル構築を行う。
担当範囲: デザイン、コーディング、お名前.comサーバーへのデプロイ、動作確認まで
保守運用: なし（納品後の更新はスポット対応想定）

5. 開発・デプロイ手順
(1) 環境構築
npx create-next-app@latest 1msp-web --typescript --tailwind --eslint
(2) お問い合わせ機能の実装
共用サーバーではNode.js APIが動作しないため、以下の構成で実装すること。
Frontend: fetchメソッドで /api/send.php へPOST送信。
Backend: public/api/send.php を作成し、mb_send_mail 等でメール送信処理を記述。
※PHPファイルはビルド時にそのまま出力されるよう public フォルダ直下に配置すること。
送付先のメールアドレスは仮でinfo@nmdlab.jpとすること
(3) ビルド設定
next.config.js を修正し、静的エクスポートモードを有効化する。
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',
  // 画像最適化コンポーネント(<Image>)は共用サーバーでエラーになるため、
  // unoptimized: true にするか、<img>タグを使用すること。
  images: {
    unoptimized: true,
  },
};
module.exports = nextConfig;
(4) デプロイ（お名前.comサーバー）
npm run build を実行。
生成された out ディレクトリの中身を、サーバーの公開ディレクトリ（例: public_html）直下にFTPソフト等でアップロード。
重要: アップロード後、必ずお問い合わせフォームの送受信テストを行うこと（PHPが動作しているか確認）。
6. 注意事項
画像パス: 静的エクスポート時、画像のパス切れが起きやすいため、相対パスやベースパスの設定に注意すること。
404エラー: 共用サーバーでは、存在しないURLにアクセスした場合の挙動がWebサーバー設定（.htaccessなど）に依存する。トップページ以外へのダイレクトアクセスは今回想定しないが、念のため確認しておくこと。
